DNS — Domain Name System

How DNS Works for This Site

When you visit network-project-self.vercel.app, your browser doesn't know where it lives on the internet. It only has the domain name. DNS translates that domain into an IP address your browser can connect to. Here's exactly what happens for this site:

1
Browser Cache CheckBrowser checks if this domain was recently resolved locally
↓
2
Recursive ResolverQuery sent to the ISP's recursive DNS resolver (in this case, 2600:6c5a:557f:9314::1)
↓
3
Root → TLD → AuthoritativeResolver walks the DNS tree: root server → .app TLD nameserver → Vercel's authoritative nameservers
↓
4
IP Addresses ReturnedVercel's DNS returns two A records: 216.198.79.67 and 64.29.17.67

The "non-authoritative" label means the response came from a cached copy at the resolver, not directly from Vercel's nameserver — this is normal and expected for frequently-accessed domains.
02

IP Addressing

IPv4 Addressing for This Site

Once DNS resolves the domain, the browser connects to the server using an IP address. This site resolves to two IPv4 addresses, which Vercel uses for load balancing and redundancy — if one server becomes unavailable, traffic is routed to the other.

IPv4 — Primary 216.198.79.67

32-bit address written in dotted-decimal notation. Vercel uses anycast routing, meaning this single IP maps to the nearest edge server to the user for low latency.

IPv4 — Secondary 64.29.17.67

A second A record provides failover capability. If the primary server is unreachable, DNS round-robin directs the browser to this alternate address.

IPv6 Status

The dig AAAA query returned no results, meaning this deployment currently does not have an IPv6 (AAAA) record configured. The site is accessible exclusively via IPv4. While IPv6 adoption is growing to address IPv4 address exhaustion (~4.3 billion addresses), many CDN providers still serve content primarily over IPv4.

What this shows: The A record query returns two IPv4 addresses, confirming load-balanced DNS resolution. Both IPs point to Vercel's edge network infrastructure.

What this shows: No AAAA record exists for this domain, meaning the site is currently IPv4-only. This is common for Vercel's default subdomain deployments. IPv6 support is typically available when using a custom domain with compatible DNS providers.

HTTP/HTTPS Protocols

How This Site Uses HTTPS

This site is served exclusively over HTTPS (port 443) using HTTP/2. Any request made over plain HTTP (port 80) is immediately redirected with a 308 Permanent Redirect to the HTTPS version. This ensures all data exchanged between the browser and Vercel's servers is encrypted via TLS.

04

Security Implementation

How HTTPS Protects This Site

Vercel automatically provisions a TLS certificate (via Let's Encrypt) and enforces HTTPS on all deployments. This means every connection to this site is encrypted, protecting against eavesdropping, data tampering, and man-in-the-middle (MITM) attacks.

TLS Handshake for This Site

1
Client HelloBrowser sends supported TLS versions & cipher suites to Vercel
↓
2
Server HelloVercel selects TLS 1.3, responds with its Let's Encrypt certificate
↓
3
Certificate VerificationBrowser validates the certificate chain against trusted Certificate Authorities
↓
4
Key ExchangeECDHE key exchange establishes a shared session key
↓
5
Encrypted SessionAll subsequent data encrypted with AES-256-GCM symmetric encryption

Security Headers Active on This Site

✓
Strict-Transport-Security (HSTS)

max-age=63072000; includeSubDomains; preload — Forces all browsers to use HTTPS for approximately 2 years. The includeSubDomains directive extends this to all subdomains, and preload allows inclusion in browser HSTS preload lists.

✓
Access-Control-Allow-Origin: *

CORS header allowing cross-origin requests. Appropriate for a public static site since all content is already publicly accessible.

✓
Cache-Control: public, max-age=0, must-revalidate

Ensures browsers always check with the server for the latest content version, preventing stale content delivery while still allowing CDN caching.
