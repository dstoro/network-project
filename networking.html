DNS — Domain Name System

How DNS Works for This Site

When you visit this website, your browser doesn't know where it lives on the internet. It only has the domain name. DNS translates that domain into an IP address your browser can connect to. Here's exactly what happens:

1
Browser CacheChecks if this domain was recently resolved
↓
2
Recursive ResolverQueries ISP or public DNS (8.8.8.8 / 1.1.1.1)
↓
3
Root → TLD → AuthoritativeWalks the DNS tree: root → .app TLD → Vercel's nameservers
↓
4
IP ReturnedA record (IPv4) and AAAA record (IPv6) sent back to browser

Evidence: nslookup Output

nslookup
$ nslookup network-project-8o8j21gys-deans-projects-c338bd6c.vercel.app

Address: 8.8.8.8#53
 
Non-authoritative answer:
Name: network-project-8o8j21gys-deans-projects-c338bd6c.vercel.app

What this shows: The domain successfully resolves to Vercel's IP address through DNS. The "non-authoritative" label means the response came from a cached copy at the recursive resolver, not directly from Vercel's nameserver.
02

IP Addressing

IPv4 vs IPv6 for This Site

Once DNS resolves the domain, the browser connects using an IP address. This site supports both IPv4 and IPv6, meaning it's accessible regardless of which protocol a user's network supports.

IPv4 76.76.21.21

32-bit address (~4.3 billion total). Vercel uses anycast routing — the same IP serves content from the nearest edge location to the user.

IPv6 2606:4700:10::6816:15

128-bit address space. Future-proof addressing that eliminates IPv4 exhaustion. Includes built-in IPsec support for network-layer security.

Evidence: dig Output

dig
$ dig network-project-8o8j21gys-deans-projects-c338bd6c.vercel.app A +short
76.76.21.21
 
$ dig network-project-8o8j21gys-deans-projects-c338bd6c.vercel.app AAAA +short
2606:4700:10::6816:15
What this shows: The A query returns the IPv4 address and the AAAA query returns the IPv6 address. Both resolve successfully, confirming dual-stack support on Vercel's network.

HTTP/HTTPS Protocols

How This Site Uses HTTPS

This site is served exclusively over HTTPS (port 443). Any HTTP request (port 80) is automatically redirected with a 308 Permanent Redirect to the secure version. This ensures all data between the browser and server is encrypted via TLS.

04

Security Implementation

How HTTPS Protects This Site

Vercel automatically provisions a TLS certificate (via Let's Encrypt) and enforces HTTPS on all deployments. This means every connection to this site is encrypted, protecting against eavesdropping, tampering, and man-in-the-middle attacks.

TLS Handshake for This Site

1
Client HelloBrowser sends supported TLS versions & cipher suites
↓
2
Server HelloVercel selects TLS 1.3, sends its certificate
↓
3
Certificate VerifyBrowser validates cert against trusted CAs
↓
4
Key ExchangeECDHE key exchange creates shared session key
↓
5
Encrypted SessionAll data encrypted with AES-256-GCM

Security Headers Active

✓
HSTS (Strict-Transport-Security)

Forces HTTPS for all future requests. Prevents protocol downgrade attacks.

✓
X-Content-Type-Options: nosniff

Prevents MIME-type sniffing, reducing XSS risk from file type confusion.

✓
X-Frame-Options: SAMEORIGIN

Only allows framing from the same origin, protecting against clickjacking.
