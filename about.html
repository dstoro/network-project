Setup Steps

01
Repository Creation

Created a GitHub repository named YOUR_USERNAME.github.io to leverage GitHub Pages' automatic deployment.

02
Website Development

Built the site using HTML5, CSS3, and vanilla JavaScript. Designed with a dark theme and terminal-inspired UI to reflect the networking/cybersecurity theme.

03
GitHub Pages Deployment

Pushed code to the main branch. Enabled GitHub Pages in repository settings (Settings → Pages → Source: main branch).

04
HTTPS Configuration

Enabled "Enforce HTTPS" in GitHub Pages settings. GitHub automatically provisions a Let's Encrypt TLS certificate.

05
DNS Verification

Ran nslookup, dig, and curl commands to verify DNS resolution, IP addressing, and HTTPS enforcement.

Challenges & Solutions

Challenge DNS propagation delay after initial deployment
Solution Waited for GitHub's DNS to propagate (~10 minutes). Verified resolution using nslookup with Google's public DNS (8.8.8.8) to bypass local cache.
Challenge Understanding the difference between A records (IPv4) and AAAA records (IPv6)
Solution Used dig with specific record type flags (A and AAAA) to query each separately, confirming dual-stack support on GitHub's infrastructure.
Challenge Verifying that HTTPS is properly enforced and not just available
Solution Tested by running curl -I http://YOUR_USERNAME.github.io (HTTP, not HTTPS) and confirmed a 301 Moved Permanently redirect to the HTTPS version, proving enforcement.
