Setup Steps

01
Repository Creation

Created a GitHub repository (network-project) for version control and continuous deployment integration with Vercel.

02
Website Development

Built a three-page site using HTML5, CSS3, and vanilla JavaScript. Dark theme with terminal-inspired UI reflecting the networking/cybersecurity subject matter.

03
Vercel Deployment

Connected the GitHub repo to Vercel. Every push to main triggers an automatic rebuild and deployment to the edge network.

04
HTTPS Enforcement

Vercel automatically provisions a Let's Encrypt TLS certificate and enforces HTTPS on all routes. No manual configuration needed.

05
Networking Verification

Ran nslookup, dig, curl -I, and openssl commands to gather evidence of DNS resolution, IP addressing, HTTPS headers, and TLS certification.

Challenges & Solutions

Challenge DNS propagation delay — site not resolving immediately after deployment
Solution Waited for Vercel's DNS to propagate (~5 min). Used nslookup with Google's public DNS (8.8.8.8) to bypass local cache and verify resolution.
Challenge Verifying HTTPS is enforced, not just available
Solution Ran curl -I http://... (HTTP, not HTTPS) and confirmed a 308 Permanent Redirect to the HTTPS version, proving enforcement.
Challenge Distinguishing between A records (IPv4) and AAAA records (IPv6) in DNS output
Solution Used dig with explicit record type flags (A for IPv4, AAAA for IPv6) to query each separately and confirm dual-stack support.
